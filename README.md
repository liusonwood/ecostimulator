# EcoSim - 生态演替模拟器 (Ecological Succession Simulator)

EcoSim 是一个空间显式（Spatially Explicit）的植被动力学模拟系统，旨在通过数学模型模拟从地衣（Lichen）到成熟森林（Tree），乃至农田生态系统的全过程。

## 🌍 项目概述

-   **核心目标：** 模拟并可视化生态系统的演替序列（地衣 → 苔藓 → 草本 → 灌木 → 乔木 → 农作物）。
-   **作者：** 由 **liusonwood** 和 **yanyi-lin** 最初制作。
-   **项目地址：** [https://github.com/liusonwood/ecosimulator](https://github.com/liusonwood/ecosimulator)
-   **技术栈：**
    -   **前端：** Vue 3 (全局脚本版) + Tailwind CSS + Lucide Icons。
    -   **计算引擎：** 采用三级自动降级机制：**GPU (GPU.js)** > **多线程 (Web Workers)** > **串行 CPU**。
    -   **可视化：** 结合 Canvas 像素级渲染与 Chart.js 实时种群动态曲线。

## 🧬 核心仿真逻辑

模型基于以下四个核心生态学过程：

1.  **种子传播与种子库 (Dispersal & Seed Bank)：**
    -   **局部传播：** 物种根据生物量产生种子，通过高斯核（Gaussian Kernel）模拟空间扩散。
    -   **背景种子雨：** 模拟远距离随机落入的种子，确保即使局部灭绝也能重新复苏。
    -   **种子生存：** 未萌发的种子在种子库中具有一定的年存活率（s）和后续萌发率（ν）。

2.  **演替门槛与土壤成熟 (Succession Barrier)：**
    -   高等植物（如乔木）的萌发受总生物量（象征有机质积累）门槛限制。
    -   **土壤异质性：** 地图随机分布浅、中、深三种土壤深度，直接修正物种的承载力 (K)。

3.  **非对称竞争 (Asymmetric Competition)：**
    -   基于 Lotka-Volterra 竞争模型。
    -   模拟层级压制：高层物种（乔木、灌木）对底层物种（地衣、苔藓）有极强的抑制作用，反之影响微乎其微。

4.  **气候与环境波动：**
    -   **年度波动：** 每年生长参数会随气候随机波动。
    -   **气候模式：** 包含热带雨林、温带草原、寒带苔原、荒漠及**特有的农场模式**。

## 🚜 特色：农场模式 (Farm Mode)

农场模式是系统中的特殊非自然状态：
-   **初始状态：** 中心区域预置高生物量的农作物，边界生成混合植被带。
-   **动态入侵：** 农作物具有较高的生长率，但容易被演替物种入侵或因灾害迅速退化。
-   **人为干预：** 在此模式下切换或手动操作会触发特定的农田生成逻辑。

## 🚀 性能说明

模拟器会自动检测硬件环境并选择最优执行路径：
-   **GPU 模式：** 利用 GPU.js 进行并行计算，适合大规模网格。
-   **Worker 模式：** 自动开启多线程并行处理，充分利用多核 CPU 性能。
-   **调试接口：** 在浏览器控制台输入 `setMode('GPU'|'Worker'|'Serial'|'Auto')` 可手动强制切换引擎。

## 🛠 交互与干预

-   **FIRE (火灾)：** 优先杀伤草本和灌木，清空大部分种子库。
-   **VOLCANO (火山)：** 毁灭性打击，彻底重置区域生物量和种子库至裸岩状态。
-   **DROUGHT (干旱)：** 全局性生长抑制，显著降低乔木和农作物的存活率。
-   **手动干扰：** 鼠标点击网格可触发局部火灾。
